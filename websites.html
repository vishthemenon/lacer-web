Website Development

This website was created using modern web development best practices, frameworks and tools.

  • Git - Used for version control and triggering the continuous integration (CI) pipeline
  • VSCode - IDE used to write the source code
  • Sketch & Photoshop - Create graphic assets (ie. logo)
  • Pug - A high-performance template engine used to markup DOM elements
  • Sass - A superset of CSS to write expressive style markup
  • ES7/ES2017 - Newest ECMAScript standard to leverage latest Javascript capabilities available on modern browsers
  • Babel - To transpile ES7 code to JS and polyfill functions that not available on older browsers
  • Webpack 4 - Module bundler for transforming, bundling, or packaging the source code and assets to obtain the build files
  • Yarn - Node package manager to install third-party node packages
  • Bootstrap 4 - Frontend CSS framework for rapid prototyping and development of the UI

To keep with best practices, the final build files are not checked into the git repository. Instead, a CICD pipeline was set-up (using Github Actions) to automatically build from source files and deploy to Github Pages when each commit is pushed to Github. This way the source repository is not polluted with temporary build files and no additional effort/time is needed to deploy the website with each release or commit.

In order to maintain code quality, code linters were used. They look for syntax errors and inconsistencies in the code.

  • StyleLint - Linter for Sass/CSS files
  • ESLint - Linter for JS files

Domain Setup

These are the series of steps needed to setup a custom domain

Decide on an available domain name that is descriptive of the website, easy to remember. You will have the choice to register the domain on several different top-level domains if available such as .com, .org, .biz, and .net . Visit a domain name registrar such as namecheap.com, Domain.com and input the chosen domain name. Ensure that the chosen registrar is also GDPR compliant.
To set up a www or custom subdomain, such as www.example.com or blog.example.com, you must create a CNAME file in your site's repository with your custom domain in the root folder.
Navigate to your DNS provider and create a CNAME record that points your subdomain to the default domain for your site. For example, if you want to use the subdomain www.example.com for your user site, create a CNAME record that points www.example.com to \.github.io. For more information about how to create the correct record, see your DNS provider's documentation.

What makes a good website?

The website of a company is very important as it is often the face of the company and plays a key part int he branding, marketing and sales strategy of the company. Hence, good website principles must be followed for the success of the company.

According to Krug’s first law of usability, the web-page should be obvious and self-explanatory. When you’re creating a site, your job is to get rid of the question marks — the decisions users need to make consciously, considering pros, cons and alternatives. If the navigation and site architecture aren’t intuitive, the number of question marks grows and makes it harder for users to comprehend how the system works and how to get from point A to point B. A clear structure, moderate visual clues and easily recognizable links can help users to find their path to their aim.
The “keep it simple”-principle (KIS) should be the primary goal of site design. Users are rarely on a site to enjoy the design; furthermore, in most cases they are looking for the information despite the design. Strive for simplicity instead of complexity. From the visitors’ point of view, the best site design is a pure text, without any advertisements or further content blocks matching exactly the query visitors used or the content they’ve been looking for. This is one of the reasons why a user-friendly print-version of web pages is essential for good user experience.
Not only does it help to reduce the cognitive load for the visitors, but it makes it possible to perceive the information presented on the screen. When a new visitor approaches a design layout, the first thing he/she tries to do is to scan the page and divide the content area into digestible pieces of information.
A website with responsive web design is a site that is able to adapt the screen it is being used on, regardless of what device it is. Companies need websites that work well on a smartphone or other mobile device screens, due to the sheer amount of people who could be using the websites through smartphones or tablets. Furthermore, Google began using mobile friendliness as a ranking signal in search results so mobile-friendly sites get a boost in serach index ranking.

What is accessibility and what considerations you should make for it?

Websites should be made inclusive and as accessible as possible to people who are disabled. Websites should be developed to accomodate for assitive technologies such as screen readers that vocalise the text on each web page. If you aim for the most universally accepted technical requirements for web accessibility, the Web Content Accessibility Guidelines (WCAG) 2.0, Level AA, you can be sure you’re providing the best, most barrier-free experience possible to the disability community. These considerations include

  1. Using screen reader (sr) tags
  2. Writing semantic HTML5
  3. Using Accessible Rich Internet Applications (ARIA) tags to inform assistive technologies
  4. Selecting smart colour coding for the visually impaired or the colour blind.

What makes a good website?

GDPR aims to protect privacy and regulates what can be done with data collected. GDPR refers to laws which help protect the website owners from privacy and data breaches. It seeks to ensure responsibility and consent. In the context of GDPR, cookies must be considered as a form of a consent. It is necessary for everyone in a team to be GDPR aware, and an information audit should be carried out with the objective of ensuring GDPR compliancy.

GDPR Guidelines
This makes the user aware that in order to view the website they must consent for the website to store or retrieve any information on a computer, smartphone or tablet. The user must agree with the terms and conditions and privacy policies. Implied consent is no longer allowed (you must click to opt-in). Moreover, the option must be unbundled meaning that each individual box must only have a single statement of agreement. There must be an easy way for a user to opt out and withdraw consent.
This should tell the users what data is being collected and for what purpose. It must also mention if any third-party members can also access their data and what their intentions are with it. This should either involve a link to the third-party policies or require the relevant information to be moved to the website itself.
This expands on the cookie policy by stating how the data is collected and stored. The data stored must be accessible to the user and, hence, there should be a way of contacting the owner to consult on what data of theirs has been stored. Additionally, users must be able to request that their own stored data be deleted from the servers.
There may be no pre-ticked boxes in forms and users must consent to receiving email advertisements.
It must be ensured that payment gateways for financial transactions are GDPR compliant and their privacy policy should be accessible from the website. In addition, if the website is required to store the data before passing it onto the gateways then it must be removed after a reasonable period of time as this is personal information.
This is where third-party marketing automation software solutions are used in the website. One must ensure that these parties are also GDPR approved and do not store and use data in ways which have not been consented to. Users must be able to access this information in the privacy policy.
If the product is traded internationally it will be necessary to determine the lead data protection supervisory authority.
